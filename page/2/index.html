让mvn编译的不同分支的同一个工程在eclipse workspace共存

工作时，有时需要在这个分支上改代码或加代码、有时候又需要去另一个分支或主干去做更改。默认通过mvn:eclipse编译后导入的project，其projectName都是一样的。而eclipse本身在workspace里是通过projectName区别不同的project的，因此同一时间只能导入工程的一个版本。此时，对于上面提到的情况需要建立多个workspace，工作时在多个workpsace之间进行切换来解决（打开多个eclipse，或File->Switch Workspace）参考了网上一些资料后，总结出了以下几个通过修改mvn:eclipse时生成的projectName来解决这个问题的方法。#groupId为关键字#osx&winmvn clean eclipse:clean eclipse:eclipse -DdownloadSources=true -Declipse. addGroupIdToProjectName =true#version为关键字#osx&winmvn clean eclipse:clean eclipse:eclipse -DdownloadSources=true -Declipse.addVersionToProjectName=true#当前目录名为关键字#osxdir=${PWD##*/}echo “mvn:eclipse dir ${dir}”mvn clean eclipse:clean eclipse:eclipse -DdownloadSources=true -Declipse.projectNameTemplate=[artifactId].${dir}#win@echo offset “cd_=%cd%”:loopset “cd_=%cd_:*\=%”set “cd_tmp=%cd_:\=%”if not “%cd_tmp%”==“%cd_%” goto loopecho “mvn:eclipse dir %cd_%”mvn clean eclipse:clean eclipse:eclipse -DdownloadSources=true -Declipse.projectNameTemplate=[artifactId].%cd_%pause#SVN分支地址为关键字#osxdir=`svn info |grep URL |awk -F/ ’{print $(NF-1)}’`echo “keyword is ${dir}”mvn clean eclipse:clean eclipse:eclipse -DdownloadSources=true -Declipse.projectNameTemplate=[artifactId].${dir}感谢： 青冥@Taobao 光远@Taobao http://qa.taobao.com/?p=5047

修改了Sunpinyin的全角快捷键

Sunpinyin的全角快捷键默认为opt+space我经常会误按这个组合，导致输入变为全角，感觉很不方便因此下了源码自己修改了一下快捷键，同时修改新图标为以前的图标首先是参考的教程，如何在maces下build SunPinyin 2.0http://code.google.com/p/sunpinyin/wiki/BuildSunpinyin2onMac下面是我操作的步骤1. 安装scons下载http://prdownloads.sourceforge.net/scons/scons-1.3.1.tar.gz，并将其解压，$ cd scons-1.3.1$ sudo python setup.py install2. 从github上clone/pull最新的sunpinyin源代码$ git clone git://github.com/sunpinyin/sunpinyin.git3. 修改图标, 原有图标需要自行下载$ cp old-sunpinyin/SunPinyin.icns sunpinyin/wrapper/macos/4. 修改快捷键使用Xcode打开 sunpinyin/wrapper/macos/SunPinyin.xcodeproj打开 工程中的 Resources/MainMeneu.xib在编辑页面左侧选择 Menu, 然后在出现的Menu中,双击全角切换右侧的快捷键此时就可以自行设置快捷键, 我的选择是直接按CMD取消了此处的快捷键不要忘了保存5. 使用scons来构建sunpinyin的工具及数据文件$ cd sunpinyin$ scons这里在编译时, 会下载几个资源文件。网络不稳定容易下载失败, 此时可以手动将链接复制到浏览器,进行下载。然后将下载后的文件放在 sunpinyin/build目录下重新运行scons命令就可以了6. 最后就是build基于IMKit的输入法前端了，$ cd wrapper/macos$ make clean; make$ sudo make install现在，sunpinyin-2.0应该已经安装到系统中了，需要re-login才可以看到这个输入法。
